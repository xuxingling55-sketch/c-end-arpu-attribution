# 4 月 C 端 ARPU 环比下滑归因分析

## 一、问题定义

本次只解释一个问题：**为什么 2026 年 4 月 C 端 ARPU 较 3 月下降明显？**

统计周期为 **2026 年 3 月 vs 2026 年 4 月**。

ARPU 口径为：

**ARPU = C 端活跃用户池内正价营收 / C 端活跃用户数**

进一步拆成：

**ARPU = 付费率 × ARPPU**

其中：

- **付费率**：C 端活跃用户中，有多少人在当月发生正价支付。
- **ARPPU**：已付费用户的人均正价营收。
- 本次归因重点看环比，不看同比。

## 二、整体表现：收入降幅远大于活跃降幅

| 指标 | 3 月 | 4 月 | 环比 |
|---|---:|---:|---:|
| C 端活跃用户 | 277.17 万 | 253.84 万 | -8.4% |
| 付费用户 | 2.87 万 | 2.00 万 | -30.4% |
| 正价营收 | 3,346.57 万 | 2,421.01 万 | -27.7% |
| ARPU | 12.07 元 | 9.54 元 | -2.54 元 |

**描述性分析：**

4 月活跃用户确实下降了，但只下降 **8.4%**。如果只是分母变小，ARPU 理论上不一定会下降，甚至可能被托住。

真正的问题在分子：正价营收下降 **27.7%**，远高于活跃用户降幅。也就是说，4 月不是“人少了一点”导致 ARPU 下滑，而是**活跃用户池里的收入产出能力明显变弱**。

## 三、公式拆解：主因是付费率下降，不是客单价下降

| 指标 | 3 月 | 4 月 | 变化 |
|---|---:|---:|---:|
| 付费率 | 1.04% | 0.79% | -0.25pct |
| ARPPU | 1,165.89 元 | 1,212.44 元 | +46.55 元 |

按公式拆解：

| 因素 | 对 ARPU 影响 | 解释 |
|---|---:|---|
| 付费率下降 | -2.90 元 | 活跃用户中买单的人变少，是主要拖累 |
| ARPPU 提升 | +0.37 元 | 买单用户客单价提升，对 ARPU 有托底 |
| 合计 | -2.54 元 | 与实际 ARPU 降幅一致 |

**描述性分析：**

4 月并不是“卖便宜了”。相反，付费用户的人均贡献还从 **1,165.89 元**提升到 **1,212.44 元**。

ARPU 下降的核心原因是：**更多活跃用户没有转化为付费用户**。付费率下降带来的负向影响达到 **-2.90 元**，远大于客单价提升带来的 **+0.37 元**。

## 四、用户客群：续费用户和老未是主要拖累

| 客群 | 付费率变化 | ARPPU 变化 | 对整体 ARPU 贡献变化 | 描述性分析 |
|---|---:|---:|---:|---|
| 续费用户 | 5.97% → 4.45% | 963 → 983 | -1.06 元 | 高效率客群转化变弱，是最大拖累之一 |
| 老未 | 0.78% → 0.59% | 1,211 → 1,264 | -1.03 元 | 活跃盘最大，转化下降会被规模放大 |
| 新增 | 0.53% → 0.45% | 1,323 → 1,328 | -0.40 元 | 首购转化下降，但影响小于前两类 |
| 高净值用户 | 0.56% → 0.47% | 1,939 → 2,036 | -0.06 元 | 客单高但规模小，对整体影响有限 |

**描述性分析：**

续费用户本来是转化效率最高的人群，3 月付费率接近 **6%**，4 月降到 **4.45%**。这类用户客单价没有下降，但买单人数明显减少，所以对 ARPU 拖累最大。

老未的单体转化率不高，但活跃规模接近整体一半。它的付费率从 **0.78%** 降到 **0.59%**，看似只降了 **0.19pct**，但由于基数大，对整体 ARPU 的拖累几乎和续费用户相当。

所以客群侧结论是：**4 月不是某个小客群异常，而是续费用户和老未两个主力经营池同时转化变弱。**

## 五、渠道归因：商业化和电销都在拖累，但问题形态不同

| 渠道 | 付费用户变化 | 付费率贡献变化 | ARPPU 变化 | 营收变化 | 对整体 ARPU 贡献变化 |
|---|---:|---:|---:|---:|---:|
| 商业化 | 19,663 → 13,106（-6,557） | -0.193pct | 431 → 397 | -325.82 万 | -1.00 元 |
| 电销 | 9,939 → 7,416（-2,523） | -0.066pct | 2,515 → 2,562 | -599.74 万 | -1.53 元 |

**描述性分析：**

商业化的问题更偏向**转化人数下滑**。3 月商业化付费用户为 **19,663 人**，4 月降到 **13,106 人**，减少 **6,557 人**，是付费人数下降最多的渠道。同时商业化 ARPPU 从 **431 元**降到 **397 元**，说明商业化不仅买单人数少了，成交结构也更偏低。

电销的问题更偏向**高客单收入减少**。电销付费用户减少 **2,523 人**，人数降幅小于商业化，但电销 ARPPU 高，4 月仍有 **2,562 元**。因此即使人数少降一些，营收仍减少 **599.74 万**，对整体 ARPU 的拖累达到 **-1.53 元**，高于商业化。

所以渠道侧结论是：**不是某一个渠道单独异常，而是商业化和电销同时走弱。商业化主要拖累付费人数，电销主要拖累高客单收入。**

## 六、商品归因：小初品、同步课、小学品承接变弱

| 客群 | 商品类目 | 营收变化 | 客群内 ARPU 贡献变化 |
|---|---|---:|---:|
| 续费用户 | 多学段商品 / 小初品 | -132.70 万 | -6.40 |
| 续费用户 | 零售商品 / 同步课 | -126.54 万 | -5.52 |
| 老未 | 零售商品 / 同步课 | -105.74 万 | -0.68 |
| 老未 | 多学段商品 / 小初品 | -87.46 万 | -0.58 |
| 新增 | 多学段商品 / 小初品 | -53.84 万 | -0.50 |
| 新增 | 单学段商品 / 小学品 | -40.67 万 | -0.34 |

**描述性分析：**

商品侧不是单一商品问题，而是几个核心承接品类同时走弱。续费用户主要掉在小初品和同步课；老未同步课和小初品都在掉；新增的小初品、小学品、同步课也都有下滑。

## 七、上新策略校正：不是断档，是切品后的承接不足

| 商品 | 3 月 → 4 月营收 | 变化 |
|---|---:|---:|
| 老品：初中规划提分课（八年级版） | 457.33 万 → 0 | -457.33 万 |
| 老品：初中规划提分课（七年级版） | 383.84 万 → 0 | -383.84 万 |
| 新品：「七升八年级」初中规划提分课 | 0 → 356.74 万 | +356.74 万 |
| 新品：「八升九年级」初中规划提分课 | 0 → 325.23 万 | +325.23 万 |
| 新品：「新高二」高中规划提分课 | 0 → 239.21 万 | +239.21 万 |
| 新品：「小升初」初中规划提分课 | 0 → 159.41 万 | +159.41 万 |

**描述性分析：**

4 月不是规划提分课完全没卖，而是商品名从“七年级版 / 八年级版”切换为“七升八 / 八升九”等升年级商品。但从 ARPU 结果看，新品虽然有承接，但**没有完全覆盖 3 月老品贡献**。

## 八、最终归因链路

**活跃用户小幅下降 → 付费用户大幅下降 → 付费率下降成为主因 → 续费用户和老未转化走弱 → 商业化付费人数减少、电销高客单收入减少 → 小初品、同步课、小学品承接不足 → 上新策略虽有新品承接但未完全覆盖老品贡献 → 正价营收下降幅度大于活跃下降 → ARPU 下滑。**

一句话版本：

> 4 月 ARPU 下滑不是客单价问题，而是转化问题；核心发生在续费用户和老未两个主力客群。渠道侧商业化主要掉付费人数，电销主要掉高客单收入；商品侧表现为小初品、同步课、小学品以及升年级新品承接不足。

## 九、口径说明

活跃用户来自 `aws.business_active_user_last_14_day`；营收来自 `dws.topic_order_detail`，并限制在同月 C 端活跃用户池内。

订单筛选条件：

- `is_test_user = 0`
- `u_user is not null`
- `original_amount >= 39`
- `business_gmv_attribution in ('电销','商业化')`
